****************************
v3.1.1 (March 11, 2026)
****************************

Bug Fixes
---------
* Fixed floating-point imprecision in nanosecond calculations in
  ``normalization.py`` by rounding the results of ``total_seconds() * 1e9``.
  On older pandas versions (e.g. 1.4.4), the float multiplication produced
  values like ``60000000000.00001`` instead of ``60000000000``, causing gap
  detection to incorrectly flag all data points as gaps. Affected functions:
  ``_t_step_nanoseconds``, ``energy_from_power``, and ``_aggregate``.
  (:pull:`494`)
* Updated expected test values in ``test_clearsky_pvlib_analysis`` and
  ``test_sensor_clearsky_pvlib_analysis`` to match corrected normalization
  output, with wider CI tolerance to account for bootstrap nondeterminism.

Requirements
------------
* Bumped minimum ``pvlib`` from 0.11.0 to 0.13.1.
* Bumped minimum ``xgboost`` from 1.6.0 to 1.7.0.post0.
* Bumped minimum ``arch`` from 5.0 to 5.6.

Testing
-------
* Enabled minimum-dependency CI testing for Python 3.10 in
  ``pytest.yaml`` (previously excluded for all Python versions).
